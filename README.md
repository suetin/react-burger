# Стартер на TypeScript для проекта Stellar Burger

## Что сделано в первом спринте

В рамках первого спринта реализован первый экран приложения Stellar Burger:

- подключена загрузка ингредиентов из API `https://new-stellarburgers.education-services.ru/api/ingredients`;
- сверстаны компоненты `BurgerIngredients` и `BurgerConstructor` с использованием UI-библиотеки;
- добавлены модальные окна `Modal` и `ModalOverlay`;
- реализованы детали ингредиента и тестовое окно заказа `OrderDetails`;
- настроено закрытие модальных окон по крестику, клику на overlay и клавише `Esc`;

## Что сделано во втором спринте

- подключены Redux Toolkit и React DnD;
- ингредиенты, модальные окна, конструктор и заказ перенесены в Redux;
- реализовано перетаскивание ингредиентов в конструктор;
- добавлены удаление, сортировка, счётчики ингредиентов и итоговая цена;
- заказ создаётся через API;

## Что сделано в третьем спринте

1. Настроена маршрутизация, страницы вынесены в директорию `pages`, а главная страница привязана к маршруту `/`.
2. Добавлен маршрут `/ingredients/:id` с модальным окном и отдельной страницей ингредиента.
3. Созданы страницы входа, регистрации, восстановления и сброса пароля.
4. Добавлены профиль, история заказов, лента заказов и страница 404.
5. Настроены активные ссылки и вложенные маршруты профиля через `Outlet`.
6. Реализованы регистрация и авторизация через API с сохранением токенов.
7. Добавлены обновление access-токена, повтор защищённых запросов и выход из системы.
8. Настроены защищённые маршруты, проверка авторизации и возврат пользователя на исходный маршрут после входа.
9. Реализовано редактирование, отмена изменений и сохранение данных профиля.

## Что сделано в четвертом спринте

Проект изначально был написан на TypeScript. В четвертом спринте проверена типизация компонентов, Redux-хранилища и утилит.

## Процедура создания коммита с проверками

При создании коммита автоматически запускаются проверка линтерами `stylelint`, `eslint` и форматирование `prettier`.

Если линтер обнаружит ошибки:

- Коммит создан не будет
- Необходимо исправить все выявленные проблемы
- Добавить исправленные файлы в индекс: `git add .`
- Повторить коммит

**Рекомендация:**  
Для предотвращения ошибок на этапе коммита предварительно выполните:  
`npm run lint`. Это позволит выявить и исправить проблемы до создания коммита.

Есть также команды для запуска проверок по отдельности:

- `npm run eslint` - запускает проверку линтера и исправляет проблемы, которые можно исправить автоматически,
- `npm run stylelint` - запускает проверку литера для файлов стилей и исправляет проблемы, которые можно исправить автоматически,
- `npm run prettier` - исправляет ошибки форматирования кода.

Можно добавить автозапуск этих команд при сохранении файла в вашей IDE.

Для создания коммита рекомендуется запускать команду `npm run commit`. Она позволяет обеспечить соответствие описаний коммитов [общепринятым соглашениям](https://www.conventionalcommits.org/en/v1.0.0/).

В проекте настроены алиасы, которые можно использовать при импорте модулей:

```
	alias: {
		'@': path.resolve(__dirname, './src'),
		'@components': path.resolve(__dirname, './src/components'),
		'@services': path.resolve(__dirname, './src/utils'),
		'@pages': path.resolve(__dirname, './src/pages'),
		'@utils': path.resolve(__dirname, './src/utils'),
	},
```
